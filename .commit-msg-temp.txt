fix(skill+ui): 小舞无敌金身被动技 hover 隐藏 + 修士克制关系不生效 双修复

────────────────── Bug 1：小舞·无敌金身 hover 隐藏 ──────────────────
现象：小舞·无敌金身是被动技（被攻击致死时自动触发并保留 1 hp），
战报已多次显示触发效果，玩家再次 hover 小舞却仍看到"效果未知（该敌方
单位尚未发动过此技能）"。

根因：S7/S7B/S7D 三个战场的 hover tooltip 一刀切：
  isEnemy && !battleRevealed → 显示"效果未知"
没区分被动 / 主动。被动技在战斗中持续生效（不需"释放"动作），所以
revealed 标记永远不会被设上，desc 被永久遮蔽 → 误导玩家。

修复：3 个 UI 文件统一加判断：
  isEnemy && !battleRevealed && !bCheck?.isPassive ? 遮蔽 : 显示
绝技字段同步加 !uCheck?.isPassive。被动技永远展示 desc，主动技仍按
"是否释放过"决定。

受益技能：所有 isPassive=true 的被动技约 50+ 个（小舞无敌金身、薰儿
古族血脉·共鸣、古元古族天火阵、奥斯卡香肠等）。

────────────────── Bug 2：修士类型克制关系不生效 ──────────────────
现象：云雀子(法)攻击立飞宇(剑)，符合 法→剑 克制关系，预期 1+1=2 点
伤害（保底1+克制+1），实际只造成 1 点。

根因：3 个 Store/Engine 的伤害结算流程顺序为
  ① 加减项 → 克制+1 → ② 翻倍 → ⑥ cap → ⑦ max(1, …)
克制 +1 在保底 max(1,…) 之前。当攻方修为 ≤ 守方修为，
(aSum-dSum) 为负，加上克制+1 后仍可能 ≤ 0，被 max(1,…) 吃掉。

修复：3 处 Store/Engine 同步改为
  ① 加减项 → ② 翻倍 → ⑥ cap → ⑦ max(1, …) → 克制+1
克制成为"无视保底的额外伤害"，但仍受 cap（cap 通常用于 hp_floor 类
保护，与克制不冲突）。对正常情况（攻击高于防御）伤害值无影响；只在
被保底兜底场景下显现 +1 差异（用户云鹊子→立飞羽案例）。

修改文件（3 个 Store/Engine + 3 个 UI + 1 个测试）：
  - src/stores/battleStore.ts          (剿匪 S7)
  - src/stores/s7bBattleStore.ts        (大比 S7B/S7C)
  - src/utils/s7dAttackEngine.ts        (决战 S7D)
  - src/systems/battle/resolveAttack.ts (普攻通用伤害管线)
  - src/screens/S7_Battle.tsx           (剿匪 hover tooltip)
  - src/screens/S7B_Battle.tsx          (大比 hover tooltip)
  - src/screens/S7D_Battle.tsx          (决战 hover tooltip)
  - src/systems/battle/__test__/stageK_liveBugFix.test.mjs
    （regex 同步：克制+1 后置后，phase ① 区域结束标志由 "// 克制"
     改为 "// 外部传入的 skillMod"，向下兼容旧布局）

────────────────── 教训沉淀 ──────────────────
"克制+1"在伤害结算的位置是关键设计决策。挪到 max(1,…) 之后可保证
"克制无视保底"，但任何后续在 max(1,…) 之后插入的钩子都需要明确
是否要叠加在克制之上（目前 __final_damage__ entry 仍在最后写入，
格式不变）。
