fix(combat): 克制+1 被保底吃掉 + 小舞被动技 hover 永久"效果未知"

【问题1：修士克制保底吃掉】
症状：云鹊子(法)6 vs 立飞羽(剑)7，战报"[克制+1] → 1点伤害"（用户期望2点）
根因：旧伤害公式 damage = (aSum-dSum) + counter → max(1,d)
      6-7+1 = 0 → max(1,0) = 1，克制被保底兜底吃掉
修复：counter 移到 Math.max(1, d) 之后
      6-7 = -1 → max(1,-1) = 1 → +counter = 2 ✅
      正常情况(aSum-dSum>0)行为不变
      克制成为"无视保底的额外伤害"，用户期望符合
影响场景：[剿匪 S7] [大比 S7B/S7C] [决战 S7D] 全战场
修改文件：
  - src/stores/s7bBattleStore.ts
  - src/stores/battleStore.ts
  - src/utils/s7dAttackEngine.ts
  - src/systems/battle/resolveAttack.ts
  - src/systems/battle/__test__/stageK_liveBugFix.test.mjs（同步 regex）

【问题2：小舞无敌金身 hover 永远"效果未知"】
症状：战报已显示"无敌金身生效"，但 hover 小舞仍显示"该敌方单位尚未发动过此技能"
根因：UI 的 revealedEnemySkills 集合只监听 lastSkillEvent（主动技/绝技释放）
      on_damage_calc 类被动 hook 写战报时从不更新 lastSkillEvent
      所以被动技永远不会"被揭示"，hover 永远遮蔽
修复：3 个战场 UI 的 hover 面板，遮蔽条件追加 && !bCheck?.isPassive
      被动技无法主动释放，遮蔽 desc 没有意义；战报已经持续展示效果
受益技能：所有 isPassive=true 的技能（无敌金身 / 古族血脉·共鸣 / 七十二路 / 焚决·噬焰 等）
修改文件：

  - src/screens/S7B_Battle.tsx
  - src/screens/S7D_Battle.tsx
